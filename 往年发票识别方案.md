# 往年发票识别与提示方案

> 目标：当发票开票年份 < 当前年份（如当前 2026，识别到 2025/2024）时，判定为"往年发票"。
> 在侧边栏 `sb-stats` 给出数量提示并置顶，并在文件卡片上打"往年发票"小标识。

---

## 1. 判定规则

- 当前年份：`new Date().getFullYear()`（运行时取，跨年自动生效）。
- 仅对**已解析**（`status === 'parsed'`）的发票做判定；失败 / 未解析文件不参与。
- 年份提取：从 `file.invoiceDate` 中用正则提取 4 位年份（`/(19|20)\d{2}/`），兼容 `2025-03-15`、`2025/03/15`、`2025年03月15日`、`2025.03.15` 等格式；`未知日期` 或空 → 视为无法确定，不计入往年。
- 判定：`提取年份 < 当前年份` → 往年发票。

新增工具函数（放 `frontend/src/utils.js`）：

```js
export function extractInvoiceYear(invoiceDate) {
  if (!invoiceDate || invoiceDate === '未知日期') return null
  const m = String(invoiceDate).match(/(?:19|20)\d{2}/)
  return m ? parseInt(m[0], 10) : null
}

export function isPreviousYearFile(file, currentYear = new Date().getFullYear()) {
  if (file.status !== 'parsed') return false
  const y = extractInvoiceYear(file.invoiceDate)
  return y != null && y < currentYear
}

// 返回 Map<fileKey, { year, isPreviousYear }>，与 duplicateInfo 同构，便于下传 FileList
export function getPreviousYearInfo(files) {
  const currentYear = new Date().getFullYear()
  const map = new Map()
  for (const f of files) {
    const y = extractInvoiceYear(f.invoiceDate)
    map.set(f.key, {
      year: y,
      isPreviousYear: f.status === 'parsed' && y != null && y < currentYear,
    })
  }
  return map
}
```

## 2. 置顶优先级（stats 提示区 + 文件列表物理置顶）

> **2026-07-19 修订**：经二次评审，用户确认"置顶"需与解析失败一致——**不仅 stats 提示区置顶，文件列表也要物理把往年发票顶到最前**（仅优先度低于失败）。因此已实现 `applySort` 四段分区。

优先级由高到低：**解析失败 > 往年发票 > 重复组 > 正常**。

- **`sb-stats` 提示区**：无失败文件、但有往年发票时，"N 个往年发票"占据 stats 顶部，优先于重复组提示；有失败文件时失败优先。
- **文件列表物理置顶**：`applySort`（`utils.js`）在原有 `失败 → 重复组 → 正常` 三段分区中插入"往年发票"段，顺序为 `失败 → 往年发票 → 重复组 → 正常`。所有调用点（`useSort` / `useFileOps` 解析完成 / `invoicePostProcessor`）均已把 `previousYearInfo` 传入 `applySort`，与 `duplicateInfo` 同构。同体（既是往年又是重复）文件按优先级归入"往年"段。

`Sidebar.jsx` 现有 `statsMode` / `countDisplay` 逻辑改为：

```js
const previousYearInfo = useMemo(() => getPreviousYearInfo(files), [files])
const previousYearCount = useMemo(() => {
  let n = 0
  previousYearInfo.forEach(i => { if (i.isPreviousYear) n++ })
  return n
}, [previousYearInfo])

const statsMode = hasFailedFiles ? 'failed'
  : (previousYearCount > 0 ? 'prevyear'
  : (duplicateInfo.size > 0 ? 'duplicate' : 'normal'))

const countDisplay = hasFailedFiles ? failedFilesCount
  : (previousYearCount > 0 ? previousYearCount
  : (duplicateInfo.size > 0 ? duplicateGroupCount : files.length))
```

`statsMode` 变化已接入现有的 `countPop` 动画（102–117 行），无需额外处理。

## 4. 文件卡片小标识（镜像"重复组"标识）

`FileList.jsx` 的 `FileListRow` 接收 `previousYearInfo` 并按优先级渲染：

- 失败文件（`has-failed`）：显示红色左条（现有规则，最高优先）。
- 否则若 `isPreviousYear`：显示 `prev-year-bar`（左侧条）+ `prev-year-label`（右上角标"往年发票"），用提示色。
- 否则若重复：显示 `duplicate-bar` + `duplicate-label`（橙色，现有）。

`FileCardRow` 关键改动：

```jsx
const pyInfo = previousYearInfo?.get(fileObj.key)
const isPrevYear = !!pyInfo?.isPreviousYear
// 优先级：failed > prevYear > duplicate
{!fileObj.failedFields?.length && isPrevYear && <div className="prev-year-bar"></div>}
{!fileObj.failedFields?.length && isPrevYear && <div className="prev-year-label">往年发票</div>}
{isDuplicate && <div className="duplicate-bar"></div>}
{isDuplicate && <div className="duplicate-label">重复组 {dupGroupIndex}</div>}
```

> 说明：失败文件已在 CSS 中 `display:none` 掉 `duplicate-bar/label`；往年条/标同样对非失败文件才渲染，避免与红条冲突。若一张发票**既是往年又是重复组**，按优先级显示"往年发票"标识（与 stats 优先级一致）。

## 5. 颜色方案（提示色）

项目语义色现状：红=失败、橙=重复/警告、紫=合并组、绿=成功、蓝=品牌。往年发票需一个**区别于橙/红/紫**的提示色，且符合现有现代风设计。

**推荐：青绿（teal）** —— 与现有色系不冲突，读作"温和提醒/注意"，不与"错误(红)""警告(橙)"混淆。

| 变量 | 亮色 | 暗色 |
|------|------|------|
| `--year` | `#0d9488` | `#2dd4bf` |
| `--year-soft` | `rgba(13,148,136,0.12)` | `rgba(45,212,191,0.14)` |
| `--year-bg` | `rgba(13,148,136,0.08)` | `rgba(45,212,191,0.10)` |

> 备选（若你更想要暖色或复用现有色）：
> - 琥珀金 `#b45309`/`#fbbf24`（接近重复橙，区分度低，不推荐）
> - 紫色 `#7c3aed`/`#a78bfa`（合并组已用，合并模式下会撞色，不推荐）
>
> 颜色集中在 `variables.css` 的两个主题块 + `sidebar.css` 的 `.prev-year-*`、`.sb-stat-summary-year`、`.sb-seg-btn-year` 几处，改一处即可全局调整。

## 6. 移除往年发票（镜像 removeDuplicateFiles）

`App.jsx` 新增 `removePreviousYearFiles(removeSource = false)`，逻辑镜像 `removeDuplicateFiles`：

```js
const removePreviousYearFiles = useCallback((removeSource = false) => {
  const liveFiles = filesRef.current
  const toRemove = liveFiles.filter(f => isPreviousYearFile(f))
  if (removeSource && toRemove.length > 0) {
    const paths = toRemove.map(f => f.path).filter(Boolean)
    const ipc = electronAPIRef.current?.ipcRenderer
    if (paths.length > 0 && ipc) {
      ipc.invoke('delete-files', paths).then(res => {
        if (res.failed?.length) console.warn('[removePrevYear] 部分文件删除失败:', res.failed)
      }).catch(err => console.error('[removePrevYear] 删除源文件出错:', err))
    }
  }
  const livePreview = previewFileRef.current
  if (livePreview && toRemove.some(f => f.key === livePreview.key)) cleanupPreviewUrl()
  setFiles(prev => prev.filter(f => !isPreviousYearFile(f)))
}, [cleanupPreviewUrl])
```

并把 `removePreviousYearFiles` 通过 props 传给 `Sidebar`（与 `removeFailedFiles`/`removeDuplicateFiles` 同位置）。
